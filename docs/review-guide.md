# 블로그 초안 검수 가이드 (SOP)

scatup AX 에이전트가 자동 생성한 블로그 초안을 팀이 GitHub Issue에서 검수·승인하는 절차입니다.
**발행은 절대 자동으로 하지 않습니다. 담당자가 승인한 뒤 사람이 직접 발행합니다.** (rule §1-2)

---

## 1. 준비 (한 번만)

- **권한:** 검수 팀원은 저장소 **Triage** 권한이면 충분합니다. (저장소 소유자가 초대)
  Triage는 라벨·담당자 지정·Issue 닫기가 가능하며, 코드나 시크릿에는 접근할 수 없어 안전합니다.
- **알림 받기:** 저장소 오른쪽 위 **Watch → All Activity**(또는 Custom > Issues)로 설정하면
  새 초안 Issue가 올라올 때 이메일·모바일 알림을 받습니다.

## 2. 검수할 초안 찾기

- **Issues** 탭에서 `scatup:draft` 라벨이 붙은 열린 Issue가 검수 대기 목록입니다.
- 라벨로 상태를 구분합니다:

  | 라벨 | 의미 |
  |---|---|
  | `scatup:draft` | 에이전트가 생성한 블로그 초안 (모든 초안에 부착) |
  | `trigger:scheduled` | 정기 주기(3일)로 생성됨 |
  | `trigger:rising` | 급상승 검색어로 추가 생성됨 |
  | `승인 대기` | AI 자동 검수(의료법·팩트체크·표절)를 통과함 |
  | `🚨담당자 판단 필요` | 의료법 애매/근거 부족/오정보 의심 → **반드시 사람이 확인** |

## 3. 초안 읽기

Issue 본문은 세 부분으로 구성됩니다.

1. **블로그 초안** — 제목 3안, 본문, 해시태그, 근거 문서
2. **트렌드 인사이트 리포트** — 왜 이 주제인지(급상승 토픽·감성 포인트·상위 수집 자료)
3. **검수 체크리스트** — 아래 4번에서 사용

> `🚨담당자 판단 필요` 라벨이면, 본문 하단 **"담당자 확인 필요 · 사유"** 를 먼저 확인하세요.
> 의료법상 단정적 표현(완치·100% 효과·부작용 없음 등)이나 근거 없는 수치가 있는지 특히 유의합니다. (rule §7)

## 4. 승인 / 반려

### ✅ 승인 (게시해도 좋다고 판단)

1. 본문 하단 **검수 체크리스트**의 체크박스를 확인하며 클릭
   - [ ] 사실 확인 — 본문의 수치·근거가 정확한지 확인
   - [ ] 표현 점검 — 40~50대 타깃에 맞는 말투·브랜드 톤인지 확인
   - [ ] 발행 승인 — 담당자가 확인한 뒤 직접 발행
2. 오른쪽 사이드바 **Labels** → **`승인`** 추가
3. 오른쪽/하단 **Close** → **"Close as completed"**
4. **발행:** 승인된 초안을 담당자가 CMS/블로그에 **직접 게시**합니다. (자동 발행 없음)

### ❌ 반려 (수정·재작성 필요)

1. **댓글**로 반려 사유를 구체적으로 작성 (무엇을 어떻게 고쳐야 하는지)
2. 오른쪽 사이드바 **Labels** → **`반려`** 추가
3. **Close** → **"Close as not planned"**

### 🚨 담당자 판단 필요 처리

- 이 라벨은 AI가 자동 통과시키지 않고 사람에게 넘긴 건입니다. **자동 발행 경로는 이미 차단**돼 있습니다.
- 사유를 확인해 문제가 없으면 → 표현을 다듬어 **승인**, 문제가 있으면 → **반려**.

## 5. 분담

- **자동 배정은 하지 않습니다.** 팀원이 Issue 목록에서 각자 맡을 초안을 골라 검수합니다.
- 중복 검수를 피하려면 검수를 시작할 때 오른쪽 **Assignees**에 본인을 지정하세요(선택).

## 6. 자주 묻는 질문

- **왜 예전 같은 승인/반려 버튼이 없나요?**
  GitHub Issue를 검수 창구로 쓰기 때문입니다. 버튼 대신 **라벨(`승인`/`반려`) + Close**로 결정합니다.
  로그인·권한·알림·이력이 GitHub 계정으로 자동 관리되어 여러 명이 안전하게 나눠 검수할 수 있습니다.
- **초안이 안 올라오는 날이 있어요.**
  정상입니다. 정기 초안은 3일에 한 번 생성되며, 그 사이에는 급상승 검색어가 감지될 때만 추가로 생성됩니다.
- **발행까지 자동으로 되나요?**
  아니요. 에이전트는 검수 대기 등록까지만 합니다. **발행은 항상 사람이 직접** 합니다. (rule §1-2, §10)
